Detailed Description

Logic for collecting and buffering keypad digits into a PIN.

  • This module manages an internal buffer to store up to 8 digits and handles the logic for submitting or canceling the entry.

◆ PinStatus

enum PinStatus

Represents the result of a key processing event in the PIN entry logic.

  • This status allows the caller (usually the main event loop) to determine if the FSM needs to be updated based on user input.
Enumerator
PIN_STATUS_IN_PROGRESS 

Character added, but PIN is not yet complete.

PIN_STATUS_VALID 

User submitted a PIN and it matched the MASTER_PIN.

PIN_STATUS_INVALID 

User submitted a PIN but it did not match.

PIN_STATUS_CANCELLED 

User pressed the cancel key; buffer has been cleared.

◆ pin_entry_process_key()

PinStatus pin_entry_process_key ( KeyEvent  event)

Processes a raw key event and updates the internal PIN buffer.

  • This function manages the state of the PIN entry buffer. It appends numbers, handles cancellations, and performs a string comparison against the master PIN when a submit command is received. The buffer is automatically reset on SUBMIT or CANCEL.
  • Parameters
    eventThe raw KeyEvent received from the keypad driver/parser.
    Returns
    PinStatus The resulting status of the PIN entry after this keypress.
  • Note
    This function logs attempts to syslog and handles the transition from raw hardware events to high-level logic statuses.
